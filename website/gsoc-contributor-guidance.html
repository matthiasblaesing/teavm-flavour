Flavour Proposal Guidance for GSoC Contributors

Welcome Contributors!

Thanks for considering Flavour as your GSoC organization. We are working to make web development easy and productive. With your help Flavour will become even better. We are looking forward to hearing what part of Flavour you want to help improve.

Proposal Prerequisites

Before submitting a proposal, please do the following easy steps:
  • Check out the Flavour Source Code
  • Build using Apache Maven: mvn clean install
  • Read over the recommended projects

Proposal Contents

A good proposal will include the following:
  • A brief CV listing relevant skills and projects
  • Your proposed project (it can be one from the project list or one of your own
  • If you've created your own project idea, give an overview of what you plan to change, why it will improve Flavour, and the impacted packages
  • A weekly project plan showing what you intend to accomplish and the deliverables each week.

Tips

  • Read the Flavour source
  • Make a sample application from the archetype to test your changes
  • Read the Flavour Book
  • Post on the Forums if you have questions
  • Have Fun!
